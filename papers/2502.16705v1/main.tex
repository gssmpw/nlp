\put(60.025,-160.3){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}The process of generating text is therefore done }
\put(46.025,-172.55){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}sequentially token by token. This means that if we were to use }
\put(46.025,-184.8){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}letters, generating the answer to that prompt would require 25 }

\put(46.025,-221.33){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}as the prediction of each token has a large computational cost }
\put(46.025,-233.58){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}in LLMs. However, using letters instead of tokens has also }
\put(46.025,-245.83){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}advantages. An obvious one, is that the LLM has fewer options }
\put(46.025,-258.08){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}to choose from for each prediction which reduces the }
\put(46.025,-270.1){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}complexity of the prediction as LLMs have to store and process }
\put(46.025,-282.35){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}information for each possible prediction outcome. This leads to }
\put(46.025,-294.6){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}a design trade-off between the number of distinct tokens and }
\put(46.025,-306.85){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}the number of predictions needed to answer to a prompt. The }
\put(46.025,-319.1){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}more distinct tokens the fewer the predictions but each being }

\put(46.025,-368.62){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}requires too many predictions and having a token for each word }
\put(46.025,-380.87){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}requires too many distinct tokens. Current LLMs have tens or }
\put(46.025,-392.87){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}hundreds of thousands of distinct tokens, for example GPT4o }
\put(46.025,-405.12){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}has approximately 200K distinct tokens. Since the goal is to }
\put(46.025,-417.37){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}reduce the number of predictions needed to answer a prompt, }
\put(46.025,-429.65){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}tokens are assigned to letter sequences that appear frequently }
\put(46.025,-441.9){\fontsize{9}{1}\usefont{T1}{cmr}{m}{n}\selectfont\color{color_29791}in the text. This means that, if a word is very common even if }
